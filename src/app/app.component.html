Routing highlights:

  • The Routing feature is used to load different components of the SPA web application in various URLs/paths.
  • Routing help dividing the applications into logical URL views by binding them to the components.
  • Routing in Angular allows for moving the user between multiple views of the SAP application, while updating the URL
  • It make the navigation feel like a multi-page site, so the site does not look like an SAP web application.
  • The routing module provides extensive URL capabilities such as query parameters, deep linking, directives, child routes and data services.
  • With Routing components are initialized through the URL/path parameters as opposed to through data binding.

Usage guidelines:

  • Create AppRoutingModule that contains the appRoutes constant, which defines all the paths/routes.
    Ex: { 'path':'users', component:UsersComponent }.
  • Import the AppRoutingModule in the main app.module.ts along with all components imports.
    Ex: import { AppRoutingModule } from './app-routing.module';
  • You could add any sub paths/urls using the colon notation.
    Ex: { 'path': ':id/:name', component: UserComponent }
  • You could use wildcards in the path property.
    Ex: { 'path': '**', redirectTo: 'not-found' }
  • You could aceess the URL parameters (the values after the colon) in the component by declaring the route in the constructor.
    Ex: constructor(private route: ActivatedRoute) { }
  • After you define and set up the path URLs, use the <router-outlet> component in your HTML template, to indicate that a component is expeced from the routing module.
  • The Router outlet component <router-outlet></router-outlet> is a placeholder for a components that gets filled dynamically by Angular using the routing object and depending on the current routing state.
  • Other components/view will call this route by using the <routerlink> directive,
  • <routerlink> can be passed a string or an array of elements/strings for the path related segments
  • <routerlink> catches the click on the HTML element and prevents the default click.
  • <routerlink> analyzes and parses the given value/path to match it with the value from the appRoutes setup, in order to decide which component to load for the given value/path.
  • <routerlink> path is relative to the current component location and can accept folder directory notation such as "../../users"
  • The CanActivate and CanDeactivate services determine if a route or path can be accessed or moved away from, and if the corresponding component will be rendered.
  • The Resolver service runs a code or does some preloading before a route or path is accessed and the component is rendered.
  • The params property of the activatedRoute is an observable which allows to work with an asynchronous tasks (it is unknkown when and if the code will be executed and we cannot block execution and wait for it to code runs).
